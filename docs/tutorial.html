How It Works

You (CFO)                Claude Desktop              ForecastAI MCP Server
    │                          │                              │
    │ "What's in this file?"   │                              │
    │ ─────────────────────►   │                              │
    │                          │  get_workbook_info_tool()    │
    │                          │ ────────────────────────►    │
    │                          │  {sheets, named ranges, ...} │
    │                          │ ◄────────────────────────    │
    │ "Here's what I found..." │                              │
    │ ◄─────────────────────   │                              │

Key principle: Claude reads your Excel files, reasons over them, and can write changes back — but only when you explicitly ask. Every destructive operation offers a backup first.

Your Excel files live in ~/Documents/ForecastAI/ (configurable). Drop any .xlsx file there and it is immediately accessible.

Setup Checklist

  • Claude Desktop installed and running
  • ForecastAI binary downloaded from the latest release
  • claude_desktop_config.json updated with the forecast-ai-excel server entry
  • Excel files placed in ~/Documents/ForecastAI/
  • Claude Desktop restarted after config change
  • Verify: type "list my Excel files" in Claude — you should see your files listed

Tips and Best Practices

For the CFO

  1. Always audit first. Before any board pack or investor meeting, run run_audit. A health score below 80 deserves investigation.
  2. Scenarios are always safe. The run_scenarios tool never modifies your file. Run as many as you like with zero risk.
  3. Label as you go. Every time you discover what a cell means, label it. After a few sessions you'll have a self-documenting model.
  4. Use provenance before decisions. Before acting on a number, trace it. "Where does this $2.4M come from?" is answerable in 30 seconds.
  5. Keep files in the workspace folder. Anything in ~/Documents/ForecastAI/ is automatically accessible.

For the Model

  1. One assumption, one cell. The audit tool flags hardcoded assumptions inside formulas.
  2. Label all unlabelled inputs before the board meeting.
  3. Consistent formula patterns across time columns. Breaks flag manual overrides.
  4. No broken external links. scan_external_references will catch stale links.

Working with Claude

  1. Be specific about cells. "Revenue cell" is ambiguous. Revenue!B5 is not.
  2. Ask for reasoning, not just numbers. Claude can explain why a variance exists.
  3. Chain your questions. After an audit, fix issues one by one. Each answer builds on the last.
  4. Confirm before any write. Claude always offers a backup before modifying a file.
